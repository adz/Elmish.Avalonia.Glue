# Preview and design data

The docs and samples keep design-time preview in the normal Avalonia workflow.

## What this shows

Design data is not a separate documentation toy. It is part of the normal authoring loop.

## Core shape

- Projection samples populate a projection root with a design model before the runtime host attaches.
- ElmView samples start the generated host with a design snapshot and keep the writable facade in the same shape at runtime.
- AXAML binds to the same ordinary data-context path in both cases.

## What you can do

- open the sample windows in preview without booting the full runtime
- inspect the same page structure in DevTools at design time and runtime
- keep the preview path close enough to runtime that markup edits remain trustworthy
