# Model

The sample matrix is the repository’s proving ground.

## What this shows

The same example set appears in both sample suites so the authored experience can be compared directly.

## Core shape

The matrix covers:

- static layout / HTML-equivalent
- user input / forms
- random / dice
- HTTP
- time / clock
- files
- SVG-equivalent composition

## What you can do

- compare Projection and ElmView on the same interaction patterns
- inspect the code and the observed output for each example
- see where the docs generation step captures example results
