# Understand

This section explains the shape of the repository before the API pages get specific.

## What this shows

- the shared substrate that both families rely on
- why Projection remains explicit
- why ElmView stays F#-first

## Core shape

The shared substrate holds the generic concepts:

- host lifetime management
- bindable snapshot nodes and hosts
- keyed collection patching
- projection contracts

The families then decide how much of that substrate is surfaced as named CLR viewmodels versus generated bindable nodes.

## What you can do

- compare the authored change surface for a normal screen in each family
- understand where generated glue ends and hand-authored UI shape begins
- decide when a feature wants stable mutable shells versus immutable snapshots
