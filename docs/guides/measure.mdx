# Measure

This section keeps the comparison explicit without turning the docs into a benchmark sheet.

## What this shows

The repository measures boilerplate, reviewability, runtime confidence, design-time quality, and editing ergonomics after ElmView V2 landed.

## Core shape

The main comparison pages read the same sample matrix from both families and look at:

- handwritten CLR glue
- AXAML surface
- editable-field wiring
- preview behavior
- runtime tests

## What you can do

- compare the current family tradeoffs without guessing
- use the sample matrix as a recurring measurement surface
- see the exact source files that back the comparison
