# Integrate

This repo can fit into an Avalonia app that already has a normal XAML and Elmish workflow.

## What this shows

- how the Avalonia dispatcher bridge stays thin
- how the Elmish host surface remains framework-neutral
- how Projection and ElmView can coexist without forcing a custom binding model

## Core shape

The integration story is deliberately modest:

- Avalonia-specific threading lives above the core host
- the core owns the lifetime and projection primitives
- the two authoring families plug into that substrate differently

## What you can do

- start an Elmish loop on the UI thread
- bind a projection root into a classic Avalonia view
- bind an ElmView generated host with standard `TwoWay` bindings
- keep design-time preview and runtime close enough to share markup
