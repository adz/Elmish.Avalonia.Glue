# Elmish

Elmish remains the state and behavior engine in both families.

## What this shows

The host owns lifetime, dispatch, and posting back to the UI thread. Elmish still owns init, update, subscriptions, and commands.

## Core shape

- `ElmishHost.start` and `ElmishHost.startAndBind` attach a running program to a host-managed sink.
- the `update` function still decides how the model changes
- subscriptions still stay in Elmish

## What you can do

- keep the Elmish architecture intact
- compare the authoring families without changing the application model
- separate UI shape from behavior more cleanly
